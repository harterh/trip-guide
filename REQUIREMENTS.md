# Trip Guide 旅游攻略网站需求文档

## 1. 文档信息

- 项目名称：Trip Guide
- 文档版本：v1.0
- 编写日期：2026-04-25
- 目标平台：Web 响应式网站，首期兼容桌面端与移动端浏览器
- 首期定位：面向计划中国境内旅行的用户，提供热门景点浏览、筛选、详情攻略与账号体系

## 2. 项目背景

用户在制定旅行计划时，通常需要同时比较景点特色、适合人群、最佳出行季节、交通方式、游玩时长、门票预约与避坑提示。Trip Guide 旨在用结构化卡片和详情页降低信息搜索成本，让用户快速找到适合自己的中国热门景点，并能保存浏览与后续行程规划。

首期以“中国十大热门景点”作为种子内容，后续可扩展城市攻略、行程路线、酒店交通、社区点评与收藏清单。

## 3. 项目目标

- 提供中国热门景点的结构化展示，包括景点卡片、地区筛选、人群筛选和详情攻略。
- 支持用户注册、登录、退出登录，为后续收藏、点评、行程规划打基础。
- 首期内置 10 个热门景点数据，包含位置、标签、建议出行时间、到达方式、游玩建议、适合人群等核心字段。
- 支持移动端友好浏览，保证用户在旅行途中也能快速查看攻略。

## 4. 非目标范围

- 首期不做在线购票、酒店预订、机票火车票预订。
- 首期不做 UGC 社区发帖与评论审核。
- 首期不接入真实地图导航、实时客流、实时票价和天气 API。
- 首期不提供复杂行程自动生成算法，仅为后续扩展预留数据结构。

## 5. 用户角色

| 角色 | 描述 | 核心需求 |
| --- | --- | --- |
| 游客用户 | 未登录访问者 | 浏览景点、使用筛选、查看详情、注册登录 |
| 注册用户 | 已注册并登录用户 | 查看全部内容、保存偏好、为后续收藏和行程功能预留 |
| 管理员 | 内容维护人员，首期可不开发后台 UI | 维护景点基础数据、攻略内容和筛选标签 |

## 6. 核心用户场景

1. 用户进入首页，看到中国热门景点卡片列表，并能按地区、人群和景点类型筛选。
2. 用户想带父母旅行，选择“亲子家庭”或“长辈友好”筛选，查看更适合低强度游玩的景点。
3. 用户点击“故宫博物院”卡片进入详情页，查看最佳出行时间、门票预约提示、公共交通到达方式和半日游路线。
4. 用户准备收藏或保存偏好时，系统引导登录或注册。
5. 用户注册成功后自动登录，并返回之前浏览的景点详情页。

## 7. 信息架构

| 页面/模块 | URL 建议 | 说明 |
| --- | --- | --- |
| 首页/景点列表页 | `/` 或 `/attractions` | 展示筛选器、搜索框、热门景点卡片 |
| 景点详情页 | `/attractions/:id` | 展示景点完整攻略、交通、季节、人群建议 |
| 登录页 | `/login` | 用户登录入口 |
| 注册页 | `/register` | 新用户注册入口 |
| 用户中心 | `/profile` | 首期展示账号信息，后续扩展收藏、行程 |
| 404 页 | `/404` | 无效景点或页面路径提示 |

## 8. 功能需求

### 8.1 首页/景点列表

首页应直接呈现可操作的旅行攻略浏览体验，不做纯营销落地页。

功能点：

- 展示“中国热门景点”列表，默认展示全部 10 个首期景点。
- 支持关键词搜索，搜索范围包括景点名称、城市、省份、标签和简介。
- 支持地区筛选、人群筛选和景点类型筛选。
- 支持清空筛选条件。
- 支持按推荐热度、建议游玩天数、适合出行月份排序。
- 景点卡片点击后进入详情页。
- 移动端筛选区应可折叠或使用抽屉，避免占用过多首屏空间。

验收标准：

- 未选择筛选条件时展示全部首期景点。
- 多个筛选条件同时存在时取交集结果。
- 无结果时展示空状态，并提供“清空筛选”操作。
- 景点卡片在桌面端至少支持 3 列布局，在移动端为单列布局。

### 8.2 景点卡片展示

每张景点卡片应包含：

- 景点封面图。
- 景点名称。
- 所在地区、城市和省级行政区。
- 景点类型标签，例如历史文化、自然风光、城市地标、亲子友好。
- 推荐理由，控制在 40 字以内。
- 建议游玩时长。
- 最佳出行季节或月份。
- 适合人群标签。
- 详情入口按钮。

交互要求：

- 鼠标悬停或触屏点击应有明确反馈。
- 标签数量过多时最多展示 4 个，其余以“+N”形式收纳。
- 卡片图片缺失时使用默认占位图，但不得影响布局稳定。
- 卡片整体可点击，详情按钮也可点击。

### 8.3 地区筛选

地区筛选首期采用固定分组：

| 地区 | 覆盖范围示例 | 首期景点示例 |
| --- | --- | --- |
| 华北 | 北京、天津、河北、山西、内蒙古 | 八达岭/慕田峪长城、故宫博物院 |
| 华东 | 上海、江苏、浙江、安徽、福建、江西、山东 | 外滩、西湖、黄山 |
| 华中 | 河南、湖北、湖南 | 张家界国家森林公园 |
| 华南 | 广东、广西、海南 | 桂林漓江 |
| 西南 | 重庆、四川、贵州、云南、西藏 | 九寨沟、布达拉宫 |
| 西北 | 陕西、甘肃、青海、宁夏、新疆 | 秦始皇帝陵博物院 |

筛选规则：

- 用户可选择一个或多个地区。
- 地区与人群筛选可组合使用。
- 当地区筛选结果为空时，展示空状态和推荐相邻地区入口。

### 8.4 人群筛选

人群标签首期包括：

- 初次来华/第一次国内长线游
- 亲子家庭
- 情侣/蜜月
- 长辈友好
- 学生党
- 摄影爱好者
- 徒步/户外爱好者
- 历史文化爱好者
- 城市周末游

筛选规则：

- 用户可选择一个或多个人群标签。
- 多个人群标签默认使用“任一匹配”逻辑，降低筛选无结果概率。
- 高级筛选可预留“全部匹配”开关。
- 景点详情页展示“适合人群”和“不太适合人群”说明。

### 8.5 景点详情页

详情页应包含以下信息模块：

| 模块 | 内容 |
| --- | --- |
| 顶部概览 | 封面图、名称、城市、评分/热度占位、核心标签、推荐语 |
| 基础信息 | 地址、开放时间说明、建议游玩时长、建议预算、预约提示 |
| 出行时间 | 最佳月份、建议避开时间、早晚游玩建议 |
| 到达方式 | 飞机/高铁/市内交通/自驾说明 |
| 攻略内容 | 推荐路线、必看亮点、拍照点、餐饮/补给提示 |
| 适合人群 | 推荐人群、强度说明、无障碍或长辈友好提示 |
| 注意事项 | 门票预约、天气、装备、安全、节假日客流 |
| 周边推荐 | 同城或相邻目的地，首期可静态展示 |

详情页交互：

- 登录用户可点击收藏按钮，首期可只保留前端状态或提示“功能建设中”。
- 未登录用户点击收藏时跳转登录页，登录后返回原详情页。
- 页面底部提供“返回列表”和“查看相似景点”入口。

验收标准：

- 每个首期景点均有可访问详情页。
- 详情页至少展示出行时间和到达方式。
- 当景点 ID 不存在时进入 404 页或展示明确错误状态。

### 8.6 用户注册

注册字段：

- 用户名：必填，3-20 个字符，支持中文、英文、数字、下划线。
- 邮箱：必填，需符合邮箱格式。
- 密码：必填，至少 8 位，包含字母和数字。
- 确认密码：必填，需与密码一致。
- 用户协议勾选：必填。

注册流程：

1. 用户填写注册信息。
2. 前端进行基础校验。
3. 后端检查用户名和邮箱是否已存在。
4. 注册成功后自动登录，跳转回来源页或用户中心。
5. 注册失败时展示明确错误原因。

安全要求：

- 密码必须加密存储，禁止明文保存。
- 注册接口需做频率限制，防止批量注册。
- 错误提示避免泄露过多账号枚举信息，邮箱已存在可提示“该邮箱不可用或已被注册”。

### 8.7 用户登录

登录方式：

- 邮箱 + 密码。
- 可预留第三方登录入口，但首期不必实现。

登录流程：

1. 用户输入邮箱和密码。
2. 前端校验必填和邮箱格式。
3. 后端校验账号密码。
4. 登录成功后写入会话或 token。
5. 登录失败时提示邮箱或密码错误。

登录状态：

- 顶部导航根据登录状态展示“登录/注册”或“用户头像/退出”。
- 刷新页面后应保持登录状态。
- 用户退出后清除本地登录态，并返回首页。

安全要求：

- 使用 HTTPS 环境部署。
- token 如存储在浏览器，应评估 XSS 和 CSRF 风险；推荐使用 HttpOnly Cookie。
- 登录接口需频率限制。

### 8.8 用户中心

首期用户中心功能：

- 展示用户名、邮箱、注册时间。
- 展示收藏功能占位：暂无收藏，后续支持保存景点。
- 提供退出登录按钮。

后续扩展：

- 我的收藏。
- 我的行程。
- 浏览历史。
- 旅行偏好设置。

## 10. 景点详情内容样例

### 10.1 长城（八达岭/慕田峪）

- 出行时间：春秋两季气温舒适，9-10 月能见度较好；暑期和国庆假期客流较大。
- 到达方式：北京市区可乘旅游专线、城际铁路或预约包车前往；慕田峪更适合希望避开超高客流的用户。
- 推荐路线：上午从市区出发，中午前抵达景区，选择缆车上行后步行精华段，下午返程。
- 攻略提示：穿防滑鞋，携带水和防晒用品；冬季风大，需注意保暖。

### 10.2 故宫博物院

- 出行时间：春秋体验最佳，建议上午入园；周末和节假日需提前规划。
- 到达方式：北京市内地铁与公交可达，建议结合天安门、景山公园安排半日到一日游。
- 推荐路线：午门入园，沿中轴线游览三大殿、后三宫，再根据兴趣选择珍宝馆、钟表馆或东西六宫。
- 攻略提示：门票预约规则变化较频繁，上线前需接入或链接官方预约说明。

### 10.3 秦始皇帝陵博物院（兵马俑）

- 出行时间：春秋适合西安周边游，暑期室内展区也可游览但客流较高。
- 到达方式：从西安市区可乘旅游专线、地铁加公交或打车前往临潼。
- 推荐路线：优先参观一号坑，再看三号坑、二号坑和文物陈列。
- 攻略提示：建议租讲解器或参加正规讲解，理解考古背景会显著提升体验。

### 10.4 张家界国家森林公园

- 出行时间：春夏植被丰富，秋季天气稳定；雨后可能出现云雾景观。
- 到达方式：可先抵达张家界荷花机场或张家界西站，再前往武陵源入口。
- 推荐路线：袁家界、天子山、金鞭溪可安排 2-3 天；体力有限用户应合理使用环保车和索道。
- 攻略提示：山区天气变化快，需准备雨具；热门索道和电梯可能排队。

### 10.5 九寨沟风景名胜区

- 出行时间：9-10 月彩林季观赏性强，5-6 月水量和植被状态较好。
- 到达方式：可飞抵九寨黄龙机场后转车，也可从成都乘长途交通前往。
- 推荐路线：沟内以观光车串联树正沟、日则沟、则查洼沟，根据体力选择步行栈道。
- 攻略提示：早晚温差明显；高峰期需关注门票预约和景区限流。

### 10.6 黄山风景区

- 出行时间：春秋舒适，冬季可能看到雪景和云海但路面湿滑。
- 到达方式：高铁到黄山北站后转景区交通；山上住宿适合看日出但需提前预订。
- 推荐路线：云谷寺上山，游览始信峰、北海、西海大峡谷，次日看日出后下山。
- 攻略提示：山路台阶多，行李越轻越好；雨雾天气需注意防滑。

### 10.7 杭州西湖

- 出行时间：春季花期和秋季气候较佳；夏季荷花季也有特色。
- 到达方式：杭州市区地铁、公交、步行和骑行均方便。
- 推荐路线：断桥、白堤、孤山、苏堤、雷峰塔可组合半日到一日游。
- 攻略提示：周末湖滨区域人流大，建议错峰或选择清晨游览。

### 10.8 桂林漓江

- 出行时间：4-6 月水量较好，9-10 月天气相对稳定。
- 到达方式：抵达桂林后乘游船前往阳朔，或选择兴坪段精华游。
- 推荐路线：漓江游船、兴坪古镇、阳朔西街、遇龙河骑行或竹筏可安排 1-2 天。
- 攻略提示：雨季需关注水位和航线调整；拍照用户可预留清晨或傍晚时间。

### 10.9 布达拉宫

- 出行时间：5-10 月相对适合进藏旅行；冬季游客较少但气候寒冷。
- 到达方式：可乘飞机或火车抵达拉萨，市区打车或公交到达。
- 推荐路线：布达拉宫、大昭寺、八廓街可组成拉萨文化一日游。
- 攻略提示：高原地区需预留适应时间，避免初到当天剧烈运动；门票和参观规则需提前确认。

### 10.10 上海外滩

- 出行时间：全年可游览，春秋步行体验更好；夜景建议在傍晚到夜间观看。
- 到达方式：上海市内地铁、公交、步行均便利，可与南京东路、陆家嘴轮渡联动。
- 推荐路线：南京东路步行街、外滩万国建筑群、外白渡桥、轮渡过江看陆家嘴夜景。
- 攻略提示：节假日外滩客流极高，应关注临时管控和返程交通。

## 12. API 需求建议

| 方法 | 路径 | 说明 | 登录要求 |
| --- | --- | --- | --- |
| GET | `/api/attractions` | 获取景点列表，支持关键词、地区、人群、类型、排序参数 | 否 |
| GET | `/api/attractions/:id` | 获取景点详情 | 否 |
| POST | `/api/auth/register` | 用户注册 | 否 |
| POST | `/api/auth/login` | 用户登录 | 否 |
| POST | `/api/auth/logout` | 用户退出 | 是 |
| GET | `/api/auth/me` | 获取当前用户 | 是 |
| GET | `/api/users/me` | 获取用户中心信息 | 是 |
| POST | `/api/favorites` | 收藏景点，后续扩展 | 是 |
| DELETE | `/api/favorites/:attractionId` | 取消收藏，后续扩展 | 是 |

列表查询参数：

- `keyword`：关键词。
- `regions`：地区，支持逗号分隔。
- `audiences`：人群标签，支持逗号分隔。
- `categories`：景点类型，支持逗号分隔。
- `sort`：`popular`、`duration`、`season`。
- `page`、`pageSize`：分页参数，首期 10 条也建议保留分页结构。

## 13. 页面体验要求

- 导航应始终提供首页、景点列表、登录/用户入口。
- 筛选状态应可见，用户能明确知道当前选择了哪些条件。
- 移动端卡片图片比例固定，避免加载时布局跳动。
- 详情页长内容应使用锚点导航或分段标题，方便快速跳转。
- 表单错误信息应显示在字段附近。
- 登录注册完成后应回到用户原本想访问的页面。

## 14. 权限与安全

- 公开信息：景点列表、景点详情。
- 登录可见：用户中心、收藏状态、后续个人行程。
- 管理员能力：首期可通过数据文件维护，后续再开发后台。
- 密码使用强哈希算法，例如 bcrypt 或 Argon2。
- 登录态推荐使用服务端 Session 或 HttpOnly Cookie。
- 所有写操作需增加 CSRF 或同等防护策略。
- 注册、登录、收藏接口需做频率限制和基础风控。

## 15. 内容运营规则

- 景点开放时间、门票价格、预约政策属于高频变化信息，上线前必须从官方渠道复核。
- 攻略内容应标注“建议”，避免承诺绝对游玩体验。
- 节假日、极端天气、高原反应、山区安全等风险提示必须出现在详情页。
- 景点图片需使用自有版权、授权图库或可商用素材。
- 热门景点排序应可配置，不应硬编码在 UI 中。

## 16. 首期验收清单

- 已创建景点列表页，默认展示 10 个景点卡片。
- 卡片包含封面、名称、地区、类型、人群、建议时长、最佳出行时间和详情入口。
- 地区筛选可用，并能与人群筛选组合。
- 人群筛选可用，支持多选。
- 每个景点均有详情页，并包含出行时间、到达方式、推荐路线和注意事项。
- 登录、注册、退出登录流程可用。
- 表单校验和错误提示可用。
- 移动端主要页面可正常浏览。
- 无结果、未登录、404 等基础状态可用。
